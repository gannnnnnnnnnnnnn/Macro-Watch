# update-dev-docs

Use this skill after meaningful project changes.

## Workflow

1. Read `AGENTS.md`, `docs/current_state.md`, `docs/dev_journal.md`, and the changed files.
2. Update only affected docs.
3. Keep `docs/current_state.md` short and current.
4. Append to `docs/dev_journal.md`; do not rewrite previous entries except to fill in pending validation or commit hash for the current entry.
5. Do not paste raw prompts.
6. Do not invent validation results. Record only commands that actually ran and their outcomes.

## Journal fields

- date/time
- task
- files changed
- what changed
- validation
- notes/risks
- next
- commit hash if available
